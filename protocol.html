▸ HTTP-NATIVE PAYMENTS · BASE NETWORK · x402

THE
PROTOCOL

Vellum is built on x402 — the HTTP-native payment protocol for the internet. Named after the dormant HTTP 402 status code "Payment Required", it lets any HTTP request carry real payment intent natively.

HOW IT WORKS
01PAYMENT FLOW x402 PROTOCOL

No wallets to connect manually. No pop-ups. No approvals. On-chain payments as frictionless as a REST API call — so AI agents can execute them with zero human intervention.

Every payment goes through Vellum's x402 middleware, which handles signing, routing, and on-chain confirmation on Base — Coinbase's Ethereum L2.

Confirmations take roughly 2 seconds. Fees are a fraction of a cent. Gas is abstracted away entirely.
